Overview

The external entities consisting of "Users" and the "Developer" are outside the boundary of the information system as outlined in the above context diagram. Users input data including their location, image of the fern being identified and their answers to key questions asked while the identification results information is returned to them. Meanwhile, the developer external entity instead inputs data including fern training images and fern key information while recieving identification error results.

Overview

The inputted data from the "Users" external entity is manipulated within the information system to verify their location and provide a fern match that is then returned to the user as an identification result, this process is possible due to the feeding of fern image data and fern keys from the fern database. Otherwise, the "Developer" external entity inputs data regarding training images and new fern keys to be processed and categorised and added to the fern database, with the developer then recieving processed misidentification report data.

Overview

The solution includes two actors: one primary actor (the User) and one secondary actor (the Developer). The User actor is associated with use cases such as inputting an image of their fern and answering questions about fern characteristics, These use cases are linked to include use case, "Verify Location", which validates that the User actor is within Wyndham. If the User actor is outside, the extend use case, "Display Location Warning" is triggered to inform them. Additionally, the extended use case, Report Misidentification, allows the user to provide feedback when an incorrect identification result is returned. Meanwhile, the Developer actor is associated with reading misidentification reports and providing fern key data and training images used by the system.

Use Case Descriptions

No. Use Case Notes
UC1 Input Image "User" actor inputs an image of their fern into the system for identification.
UC2 Answer Key Question "User" actor provides answers on fern characteristics into the system for identification.
UC3 Verify Location (Include Use Case) The system verifies whether the user's location is within Wyndham City.
UC4 Display Location Warning (Extend Use Case) The system displays a warning when the user's location is outside Wyndham City.
UC5 Report Misidentification (Extend Use Case) "User" actor submits a misidentification report when an incorrect fern identification result is returned.
UC6 Read Misidentification Report "Developer" actor reads misidentification reports from system.
UC7 Provide Fern Keys "Developer" actor provides fern key data for system.
UC8 Provide Training Images "Developer" actor provides fern training images for system.
